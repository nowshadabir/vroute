# vroute

vroute is a zero-config local DNS and SSL router designed for seamless local web development. It allows developers to map custom local domains (like `app.test` or `*.app.localhost`) to local development servers (e.g., port 3000), complete with valid HTTPS/SSL certificates and CORS handling.

## Key Features

1. Reverse Proxy Engine
- HTTP and HTTPS proxies that intelligently route requests based on the `Host` header.
- Automatically handles WebSocket upgrades (e.g., for Next.js or Vite Hot Module Replacement).
- Injects `Access-Control-Allow-Origin: *` to bypass CORS issues during development.

2. Cross-Platform DNS Management
- Dynamically modifies the OS `hosts` file (Linux, macOS, Windows) for exact-match domains.
- Includes a built-in Local DNS Server (UDP port 53) to support wildcard subdomains (e.g., `*.app.localhost`), resolving them dynamically to `127.0.0.1` without needing `hosts` file entries.

3. Automated Local SSL Generation
- Generates a Local Root CA on setup and automatically injects it into the OS trust store.
- Dynamically provisions per-domain or wildcard SAN certificates on the fly using `node-forge`.
- Provides a "Green Padlock" secure context for all local development servers.

4. Multi-Tenant Support
- Intercepts wildcard subdomains and automatically injects a tenant identity header (e.g., `X-Vroute-Tenant`) into the backend request, simplifying local testing for SaaS applications.

5. Web Dashboard
- A React + Tailwind based UI to easily add, manage, and remove routes.
- Includes a real-time traffic log showing HTTP methods, statuses, and response times.

## Project Structure

- `packages/vroute-cli`: The core package containing the CLI orchestrator, the background Node.js daemon (Express + http-proxy), the DNS server, the SSL generator, and the React web dashboard.
- `bin/`: Contains worker scripts used for privilege escalation (sudo) when modifying the hosts file or injecting Root CA certificates.

## Status

The core features (Core Architecture, Proxy Engine, SSL, DNS, Dashboard, Packaging, and Wildcard Subdomains) are fully implemented. Future potential work includes creating a standalone Desktop application (Electron/Tauri) with System Tray integration and route import/export capabilities.
